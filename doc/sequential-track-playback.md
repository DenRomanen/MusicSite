# Sequential Track Playback

Добавлено последовательное воспроизведение треков в библиотеке без ручного подтверждения.

- В `frontend/src/widgets/track-library/TrackLibrary.tsx` добавлен реестр `audio`-элементов по `track.id`, чтобы родительский список мог запускать следующий трек программно
- После события `ended` плеер определяет следующий трек в текущем отфильтрованном и отсортированном списке, переключает страницу при необходимости и сразу вызывает воспроизведение
- В `frontend/src/entities/track/ui/TrackListRow.tsx` проброшен `ref` текущего `audio`, чтобы библиотека могла управлять автопереходом между треками
