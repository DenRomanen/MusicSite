Сгенерируй полностью готовый fullstack-проект музыкального сайта.

Стек:
- frontend: React + Vite + React Router
- backend: Node.js + Express
- database: SQLite
- auth: JWT
- uploads: multer

Функционал:
- 2 основные страницы: "/" и "/login"
- страница 404 для всех остальных маршрутов
- главная страница доступна всем
- на главной сразу показывается список музыкальных произведений
- любой пользователь может прослушивать музыку
- в правом верхнем углу кнопка «Войти»
- зарегистрироваться нельзя, регистрации быть не должно вообще
- логин и пароль выдает только администратор
- создай seed-пользователя:
  login должен браться из env
  password должен браться из env
- после авторизации пользователь может загружать музыкальные произведения на сервер
- после авторизации пользователь может удалять только те музыкальные произведения, которые сам загрузил
- неавторизованные пользователи вообще не должны видеть кнопки добавления и удаления
- на сайте не должно быть лишней информации
- дизайн должен быть минималистичный, красивый, современный, с элементами классики

Что должно быть реализовано:
1. backend API:
   - POST /api/auth/login
   - GET /api/auth/me
   - GET /api/tracks
   - POST /api/tracks
   - DELETE /api/tracks/:id

2. база данных:
   users(id, login, password_hash, created_at)
   tracks(id, title, artist, filename, original_filename, mime_type, size, uploaded_by, created_at)

3. правила доступа:
   - GET /api/tracks доступен всем
   - POST /api/tracks только авторизованному
   - DELETE /api/tracks/:id только авторизованному и только для своих треков

4. хранение файлов:
   - папка uploads/audio
   - допустимые форматы: mp3, wav, ogg

5. главная страница:
   - список треков
   - название
   - исполнитель
   - дата добавления
   - audio player
   - для своих треков у авторизованного пользователя кнопка удаления

6. страница входа:
   - login
   - password
   - кнопка входа
   - без регистрации

7. UI:
   - минимализм
   - современность
   - легкие классические мотивы
   - чистая композиция
   - светлая нейтральная палитра
   - serif для крупных заголовков, sans-serif для остального
   - аккуратные карточки
   - адаптивность

8. проект должен содержать:
   - frontend
   - backend
   - seed-скрипт
   - .env.example
   - README
   - обработку ошибок
   - backend 404
   - frontend 404

Требования к генерации:
- не задавай уточняющих вопросов
- не описывай, а сразу генерируй код
- покажи все нужные файлы
- добавь package.json для frontend и backend
- добавь npm scripts
- в конце покажи дерево проекта и инструкцию запуска
